Hermes Agent — Orchestrator Architecture

/local/plugins & /local/scripts — Fleet Plugin/Script Distribution

Orchestrator Node

  • • Owns /local/plugins bootstrap on gateway restart
  • • Runs prestart_reapply.sh at gateway startup
  • • Applies plugin hooks to all gateway clones
  • • Distributes shared plugins from /plugins/public
  • • NODE_STATE=1 (orchestrator role)

/local/plugins Structure

  • • private/ — node-specific (discord commands, wiki, memory)
  • • public/discord/ — slash bridge, ACL, routing hooks
  • • public/hermes-core/ — wiki engine, governance, coordination
  • • public/wiki/ — shared wiki plugin template
  • • Reapplied on every gateway restart

/local/scripts Structure

  • • private/backup + crons/ — node-specific jobs
  • • public/clone/ — gateway clone lifecycle (horc/hord)
  • • public/ui-gateway/ — Flask HTTP API (7 modules)
  • • public/backup/ — Google Drive snapshot/restore
  • • public/update/ — fleet plugin matrix + apply
