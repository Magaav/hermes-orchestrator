# Commercial Direction

Hermes Orchestrator is an open-core effort to build local-first infrastructure for agent fleets, spatial workspaces, browser-backed automation, and shareable wasm-agent artifacts.

The public repository focuses on the local runtime, orchestration layer, plugin contracts, developer-facing architecture, documentation, and security posture.

The goal is to keep the local runtime useful, transparent, and extensible while building sustainable hosted infrastructure around it.

## Public Core

The public core may include:

- Hermes Orchestrator local runtime
- wasm-agent local workspace shell
- plugin contracts and extension points
- local-first artifact specifications
- developer documentation
- security posture and launch hardening
- examples, demos, and local development tooling

## Commercial Layers

Future commercial layers may include:

- wasm-agent-cloud
- hosted browser workers
- shared-space relay
- artifact hosting
- cloud sync
- managed agent nodes
- premium connectors
- enterprise deployments
- team/workspace management
- advanced observability
- private automation templates
- usage-based cloud resources powered by Flux Credits

## Flux Credits

Flux Credits are intended as prepaid, non-transferable usage credits for wasm-agent-cloud resources.

Flux may be used for resources such as:

- hosted agent runs
- browser workers
- shared-space relay
- artifact hosting
- cloud sync
- premium model usage
- storage
- managed runtime capacity

Flux Credits are not money, not crypto, not transferable, and not redeemable for cash.

## Commercial Use

This repository may contain open-source components, but commercial hosted services, managed deployments, enterprise features, billing systems, private infrastructure, and wasm-agent-cloud services may live outside this public repository.

The public runtime is designed to remain useful on its own while enabling a sustainable commercial ecosystem around hosted infrastructure, support, and managed services.
